## 歌曲文件要求

| 项目 | 要求 |
| --- | --- |
| 格式 | .wav 格式的无损音频文件 |
| 位深 | 至少 16 Bit |
| 采样率 | 至少 44.1 KHz |
| 空白音频 | 开头或结尾空白音频长度不得超过 3s |

## 封面要求

### 内容要求
- 不得出现任何违反法律的信息。
- 不得出现合同细节或社交媒体账户等信息。
- 避免显示任何形式的条码、二维码。
- 不得含有价格标签或任何与价格有关的信息。
- 禁止出现品牌标志或任何形式的广告内容。
- 图片中不应包含具体的日期信息。
- 不得提及或展示任何实物包装。
- 必须确保所使用的元素不会侵犯他人的版权。

### 格式要求
| 项目 | 要求 |
| --- | --- |
| 格式 | .jpg 格式 |
| 尺寸 | 标准正方形，分辨率不低于 3000x3000 像素 |

## 发行日期选择
- 请选择合适的发行日期，并确保专辑在该日期前后不会有**重大变动**。
- **常规发行及周末发行**，建议提前 **4 - 5 天**做好准备，此为最佳时间安排；若情况紧急，提前 **2 - 4 天**也可行。
- 若选择在**法定节假日**发行，建议提前 **7 天**准备到位，该时间安排更为稳妥。
- **都需要在工作日发送物料 周末不受理**

## 歌手信息.txt的填写要求

### 整体格式
- 文件需按照以下固定结构进行填写，保持清晰的层级关系和格式规范。

### 歌曲信息部分
- **歌曲名称**：必填项，准确填写歌曲的正式名称，如 `歌曲名称：Falling Apart`。
- **歌曲副标题/各种文翻译**：选填项，若有歌曲副标题或不同语言翻译，可在此处填写，格式为 `歌曲副标题/各种文翻译（选填）： ` 。

### 艺人信息部分
- **艺人名**：必填项，若有两个及以上艺人，艺人名称之间用 `/` 分隔，如 `艺人名（两个及以上的制作人中间加/）：Artist X/Artist Y`。
- **艺人详细信息**：针对每个艺人分别填写以下内容：
  - **艺人真实姓名**：必填项，为满足发行公司需求，同时确保艺人隐私，可填写英文名、拼音并确认，格式为 `艺人真实姓名（发行公司需要，为确保艺人隐私，可填写英文名/拼音并确认）：John Doe`。
  - **主页链接**：
    - 国内音乐平台链接（如网易云音乐、QQ 音乐、酷狗、酷我、汽水音乐）：若艺人在对应平台有主页，需填写完整链接；若无则留空，各平台链接分行填写，格式如 `网易云音乐：https://music.163.com/#/artist?id=xxxxxx`。
    - 外网音乐平台链接（如 Spotify、Youtube、Apple Music、SoundCloud）：选填项，若艺人在对应平台有主页，填写完整链接；若无则留空，各平台链接分行填写。
